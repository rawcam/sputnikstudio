Общее описание

Sputnik Studio — веб-приложение для профессионального проектирования аудиовизуальных систем. Оно позволяет рассчитывать параметры видеотрактов, сетевую нагрузку, LED-экраны, акустические характеристики, требования к видеоконференцсвязи и эргономику. Все расчёты выполняются в реальном времени, проект сохраняется в браузере и может быть экспортирован в JSON.

Интерфейс состоит из левой боковой панели (сайдбара) с аккордеонами разделов и основной рабочей области. Сайдбар можно сворачивать, оставляя только иконки, что удобно на небольших экранах.

Продукт разработан для инженеров по AV-системам, системных интеграторов, технических консультантов. Он помогает быстро оценить ключевые параметры на этапе концептуального проектирования.

Необходимая минимальная квалификация инженера

Для эффективного использования Sputnik Studio рекомендуется иметь:

  • Понимание основ передачи видеосигнала (разрешения, частоты кадров, субдискретизация, битрейт).
  • Знание сетевых технологий (Ethernet, коммутаторы, PoE, QoS, multicast).
  • Опыт проектирования LED-экранов (шаг пикселя, размеры кабинетов, разрешение).
  • Базовые знания акустики (SPL, реверберация, подбор громкоговорителей).
  • Понимание видеоконференцсвязи (кодеки, битрейт, многоточечные вызовы).
  • Умение интерпретировать результаты расчётов и применять их в реальных проектах.

Программа не требует глубокого программирования, но знание терминологии AV-индустрии обязательно.

Глоссарий

Основные термины

SPL (Sound Pressure Level) — уровень звукового давления, измеряется в дБ. Показывает громкость звука в заданной точке.

RT60 — время реверберации, за которое уровень звука в помещении снижается на 60 дБ после прекращения источника.

PoE (Power over Ethernet) — передача питания по сетевому кабелю. Стандарты: 802.3af (до 15.4 Вт), 802.3at (до 30 Вт), 802.3bt (до 90 Вт).

QoS (Quality of Service) — механизм приоритизации трафика в сетях.

Multicast — метод передачи данных от одного источника группе получателей, экономящий пропускную способность.

Headroom — запас по уровню сигнала, чтобы избежать искажений при пиковых нагрузках.

Кодек — алгоритм сжатия видео/аудио (H.264, H.265, MJPEG).

MCU (Multipoint Control Unit) — устройство для многоточечных видеоконференций.

Карточка устройства

В разделе «Тракты» каждое устройство отображается в виде карточки. Карточка может быть в развёрнутом или свёрнутом виде. Управление:

  • / — сворачивание/разворачивание карточки.
  • — удаление устройства.
  • Чекбоксы USB, PoE, Ethernet — включают соответствующие функции.
  • Поле Вт — ручное изменение потребляемой мощности.

Видео

Позволяет задать параметры видеосигнала: разрешение (1080p, 4K, 8K), субдискретизацию (4:4:4, 4:2:2, 4:2:0), частоту кадров, цветовое пространство (RGB/YCbCr) и глубину цвета (8/10/12 бит). На основе этих данных вычисляется базовый битрейт, который затем используется для оценки загрузки сети.

Формула: bitrate = base * chromaFactor * colorSpaceFactor * bitDepthFactor * (fps/60), где base = 300 (1080p), 1000 (4K), 4000 (8K).

Сеть

Выбор среды передачи (Cat5e…Cat8, оптика, беспровод), включение Multicast и QoS, тип сети (управляемая/неуправляемая), протокол синхронизации (PTP/NTP), резервирование. Эти параметры влияют на допустимую нагрузку, но в текущей версии используются для расчёта общей загрузки сети (отношение суммарного битрейта к минимальной пропускной способности коммутаторов).

LED

Три режима расчёта LED‑экранов:

  • По кабинетам – выбор шага пикселя, типоразмера кабинета (600×337.5, 500×500, 640×480, 960×540 или свой), количество кабинетов по горизонтали и вертикали. Результат: физические размеры (м), разрешение, площадь, мощность.
  • По разрешению – задаётся желаемое разрешение (Full HD, 4K, 8K или своё). Система подбирает ближайшее целое количество кабинетов (округляя вверх) и выводит реальное разрешение, размеры, количество кабинетов, площадь и мощность.
  • Сшивка экранов – позволяет объединить несколько одинаковых LED‑экранов в один логический экран. Нужно выбрать ранее созданный LED‑экран из трактов и задать количество по горизонтали и вертикали.

Звук

Пять режимов акустических расчётов:

  • SPL на расстоянии – уровень звукового давления для точечного источника с учётом чувствительности, мощности, расстояния, headroom и добавки на помещение.
  • Падение SPL от расстояния – уменьшение громкости при увеличении дистанции для точечного и линейного источников.
  • Изменение SPL от мощности – прирост уровня при увеличении подводимой мощности.
  • RT60 (реверберация) – время реверберации по формуле Сэбина (0.161 * V / (α * S)).
  • Подбор громкоговорителей – приблизительный расчёт количества потолочных колонок на основе размеров помещения, высоты потолка, мощности и чувствительности громкоговорителя, требуемого SPL.

ВКС (видеоконференции)

Расчёт требуемого битрейта для видеоконференций. Доступны два режима:

  • Кодек – выбор платформы (TrueConf, Vinteo IVA, Cisco Webex, Poly, Yealink), разрешения, FPS, количества участников в P2P. Вычисляется битрейт отправки и приёма, рекомендация по каналу.
  • Многоточечный вызов – аналогично, но для MCU‑конференции, где суммируются потоки от всех участников.

Базовые значения битрейта для 1080p@30: TrueConf – 2.5 Мбит/с, Vinteo IVA (H.265) – 1.5 Мбит/с, Cisco Webex – 2.0 Мбит/с, Poly – 2.0 Мбит/с, Yealink – 1.8 Мбит/с.

Тракты

Позволяет создавать сквозные тракты передачи сигнала от источника до приёмника. В тракт можно добавлять устройства: источники, передатчики, приёмники, коммутаторы, матрицы, LED‑процессоры, экраны и др. Для каждого устройства задаётся задержка, питание, наличие PoE, USB и другие параметры. Суммарная задержка тракта отображается в мс и кадрах. Также ведётся учёт сетевых портов, PoE‑бюджета и общей мощности.

Эргономика ЭКП

Два калькулятора:

  • Расчёт расстояния/высоты символа – по угловым размерам (α) определяет максимальное расстояние до экрана по заданной высоте символа или минимальную высоту символа по известному расстоянию.
  • Зона оптимальной видимости (ЗОВ) – позволяет задать координаты нескольких наблюдателей и оценить, какая часть строки на экране попадает в зону с углом ≥ критического. Итоговый коэффициент эргономики Кэ = (средняя доля зоны) × 10.
Формулы: h = L · tg(α) | L = h / tg(α), где α – угловой размер (0.25° – минимальная читаемость, 0.38° – оптимально, 0.55° – критическая информация).

Управление проектом

Все данные сохраняются в локальном хранилище браузера. Кнопки в разделе «УПРАВЛЕНИЕ» позволяют:

  • Сохранить в браузере – записать текущий проект в localStorage.
  • Экспорт JSON – выгрузить проект в файл для резервного копирования или обмена.
  • Импорт JSON – загрузить ранее сохранённый проект.
  • Печать отчёта – сформировать подробный отчёт в новой вкладке.
  • Сброс проекта – удалить все данные и начать заново.
